# Four-Week Calendar Plan Template

Use actual calendar availability before assigning dates. Avoid treating every open hour as focused technical time.

## Weekly Pattern

- **Short coding session:** 45 minutes
- **Deep review session:** 75–90 minutes
- **Testing/remediation session:** 75–120 minutes
- **Weekly review:** 30 minutes
- **Buffer:** 45–60 minutes

## Event Template

- **Title:** Solidity Lab — [exercise and objective]
- **Duration:** [45 / 75 / 90 / 120 minutes]
- **Repository:** `mcleanlipschutz/solidity-security-learning-lab`
- **Objective:** One measurable outcome
- **Deliverable:** Exact file or test expected
- **Prerequisites:** Required issue or exercise state
- **Related work item:** GitHub/Linear identifier after creation

## Sequencing Rule

Do not schedule proof-of-concept work before the threat model, remediation before reproduction, or report completion before regression verification.
