# Ethics and Authorized Scope

This repository is for local, educational, and explicitly authorized security practice.

Do not:

- Target production contracts or systems without written authorization.
- Interact with contracts holding real third-party funds for exploit testing.
- Publish private keys, credentials, protected answer keys, or sensitive incident data.
- Present hypothetical concerns as confirmed vulnerabilities.
- Copy public solutions before completing an independent attempt.

Proof-of-concept tests should run on local Anvil state or an authorized fork pinned to a historical block when appropriate.
